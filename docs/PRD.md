# 产品需求文档 (PRD)

> AI 智能客服系统 v1.0 | 2026-06-23

## 1. 产品概述

### 1.1 产品定位

基于大语言模型 (LLM) 的企业级智能客服系统。用户上传产品文档、FAQ、政策文件等知识库资料后，系统自动解析并向量化存储。用户提问时，通过 RAG（检索增强生成）架构检索相关知识片段，由 LLM 生成精准、有来源可追溯的回答。

### 1.2 目标用户

| 角色 | 需求 | 场景 |
|------|------|------|
| 普通用户（C 端） | 产品咨询、售后问题、政策查询 | Web 端自助问答 |
| 企业管理员（B 端） | 管理知识库文档、查看问答统计 | 上传文档、监控服务质量 |

### 1.3 核心价值

- **降本**：AI 自动回答替代人工客服，减少人力成本
- **增效**：7×24 小时服务，秒级响应
- **可信**：每回答必标注引用来源，杜绝 AI 幻觉

## 2. 功能需求

### 2.1 用户与会话模块 (P0)

#### 2.1.1 用户注册/登录

| 字段 | 说明 |
|------|------|
| 注册方式 | 手机号 + 密码（邮箱仅用于登录查找，不用于注册） |
| 密码规则 | 最少 8 位，须含大写字母+小写字母+数字+特殊字符，bcrypt 哈希存储 |
| 认证方式 | JWT Token，有效期 24 小时 |

**验收标准：**
- [ ] 手机号/邮箱格式校验
- [ ] 手机号/邮箱唯一性检查
- [ ] 登录后返回 JWT，后续请求 Header 携带 `Authorization: Bearer <token>`

#### 2.1.2 会话管理

| 功能 | 说明 |
|------|------|
| 创建会话 | 每次对话为独立 Session，默认标题"新会话" |
| 会话列表 | 按更新时间倒序，显示标题、消息数、时间 |
| 会话详情 | 查看完整对话记录（用户消息 + AI 回答） |
| 会话标题 | 自动取首条用户消息前 30 字 |

**验收标准：**
- [ ] 会话列表仅显示当前用户的会话
- [ ] 点击历史会话加载完整消息
- [ ] 新建会话后立即出现在列表顶部

#### 2.1.3 反馈

| 功能 | 说明 |
|------|------|
| 评价类型 | 赞 (positive) / 踩 (negative) |
| 可选评论 | 踩时可填写文字反馈 |
| 限制 | 每条消息仅可反馈一次 |

**验收标准：**
- [ ] 反馈后 UI 状态锁定，不可重复提交
- [ ] 踩的回答记录日志

**验收标准：**

### 2.3 智能问答模块 (P0)

#### 2.3.1 RAG 问答流程

```
用户输入问题
  → 输入校验（长度 ≤ 500 字 + 每日次数 ≤ 100）
  → 向量检索（BGE-M3 Embedding → Milvus Top-K 搜索）
  → 若检索为空 → 返回兜底话术，不调 LLM
  → 若检索有结果 → 拼接 Prompt（System + 历史 + 知识片段）
  → DeepSeek API 流式调用
  → SSE 流式输出 → 前端逐字渲染
  → 展示引用来源（文档名 + 片段摘要）
```

#### 2.3.2 业务规则

| 规则 | 默认值 | 可配置 |
|------|--------|--------|
| 单次提问长度上限 | 500 字 | ✅ |
| 每日提问次数上限 | 100 次 | ✅ |
| 检索返回片段数 (Top-K) | 12 | ✅ |
| 相似度最低阈值 | 0.55（兜底 0.35） | ✅ |
| 多轮对话携带轮数 | 5 轮 | ✅ |
| LLM 超时 | 30 秒 | ✅ |
| Token 预算 | 8000 | ✅ |

#### 2.3.3 流式输出

| 要求 | 说明 |
|------|------|
| 传输协议 | SSE (Server-Sent Events) |
| 渲染方式 | 前端逐字显示，打字机效果 |
| 事件类型 | `token` (逐字) / `sources` (引用) / `error` (异常) / `done` (结束) |

**验收标准：**
- [ ] 首 token 延迟 < 3 秒（不含模型加载时间）
- [ ] 流式输出无卡顿
- [ ] 异常时前端显示错误提示

### 2.5 AI Agent 任务拆解 (P2)

将自然语言需求自动拆解为微服务层面的改动分析任务列表。

```
用户输入: "用户下单后未收到确认短信，怀疑系统未发送通知"

Agent 输出:
  1. [high] 排查短信发送服务 — sms-service
  2. [high] 核查订单状态 — order-service
  3. [medium] 检查通知规则 — notification-service
```

**验收标准：**
- [ ] 输入需求文本，返回结构化任务列表（title/description/service/priority）
- [ ] 任务数 2-5 个
- [ ] LLM 强制 JSON 输出（`response_format={"type": "json_object"}`）

## 3. 非功能需求

### 3.1 性能

| 指标 | 目标 |
|------|------|
| API 响应时间 (非 AI 接口) | < 200ms |
| 首 token 延迟 | < 3s |
| 并发用户数 | 50+ |
| 单次问答 Token 消耗 | < 8000 |

### 3.2 安全

- [ ] 密码 bcrypt 哈希，不存储明文
- [ ] JWT 认证，失效后重定向登录
- [ ] 无硬编码 API Key
- [ ] `.env` 不入库，提供 `.env.example`
- [ ] 输入校验（长度、格式、频率）
- [ ] SQLAlchemy 参数化查询防注入
- [ ] CORS 白名单

### 3.3 可用性

- [ ] 检索为空时兜底话术，不白屏
- [ ] LLM 超时时返回友好错误
- [ ] API 限流时显示"服务繁忙"
- [ ] 网络异常时前端提示重试

### 3.4 可维护性

- [ ] 所有阈值参数通过 `.env` 配置
- [ ] RAG 模块独立于 Web 层
- [ ] 核心逻辑有单元测试
- [ ] 完整文档（API / 数据库 / AI 架构 / 业务流程 / 运行指南）

- **users** — 用户

### 5.1 数据初始化建议

为简化开发测试，系统预置 7 篇示例文档，安装时自动完成向量化：

| 文档 | 内容 | 预估字数 |
|------|------|----------|
| `公司产品介绍.txt` | 产品概述、核心功能、版本价格、技术支持 | ~800 |
| `常见问题FAQ.md` | 账号、使用、计费三大类 FAQ | ~1200 |
| `退换货政策.txt` | 退换货条件、流程、运费、退款说明 | ~1000 |
| `用户使用手册.txt` | 注册、登录、设置、快捷键等操作指引 | ~1500 |
| `API接口说明.md` | API 鉴权方式、限流规则、常见错误码 | ~1000 |
| `隐私政策.txt` | 数据收集范围、存储方式、用户权利 | ~800 |
| `服务协议.txt` | 服务条款、免责声明、争议解决方式 | ~900 |

**总计**：约 7200 字（覆盖多领域知识，便于测试多知识库路由）

**初始化方式**：执行 `python init_knowledge.py` 一键完成：
1. MySQL 建表
2. 7 篇文档自动解析→分块→BGE-M3 Embedding→Milvus 入库
3. 初始化完成后系统即可直接测试问答

**初始化后验证**：
- 提问"退换货流程是什么？"→ 应返回退换货政策相关内容
- 提问"有哪些版本？多少钱？"→ 应返回产品版本和价格表
- 提问"今天天气怎么样？"→ 应返回兜底话术（知识库无此信息）
