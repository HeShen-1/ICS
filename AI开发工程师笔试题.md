# 项目任务：AI 智能客服系统

## 核心目标

构建一个基于大语言模型（LLM）的企业级智能客服系统，实现从用户提问、意图识别、知识检索到生成回答的完整 AI 对话链路。

## 功能需求

### 1. 用户与会话模块

**· 用户端功能**：

* 用户注册/登录（手机号+密码 或 邮箱+密码）
* 发起对话会话（每次对话为独立 Session）
* 查看历史会话列表与会话详情（含完整对话记录）
* 对 AI 回答进行反馈（点赞 / 踩，可选填文字反馈）

### 2. 核心 AI 对话模块

**· 知识库管理**：

* 支持上传文档（.txt / .md / .pdf），系统自动解析并向量化存储
* 知识库列表展示（文档名称、上传时间、状态：处理中 / 就绪 / 失败）
* 支持删除知识库文档（删除后对应向量数据同步清除）

**· 智能问答流程（RAG 架构）**：

* 用户输入问题 → 向量检索相关知识片段 → 拼接 Prompt → 调用 LLM → 流式返回回答
* 每次回答需展示引用的知识来源（文档名称 + 片段摘要）
* 支持多轮对话（上下文携带最近 N 轮历史消息）

**· 业务规则**：

* 单次提问长度不超过 500 字
* 若检索不到相关知识，返回标准兜底话术，不编造答案
* 每个用户每日提问次数上限为 100 次（可配置）
* 流式输出（Streaming）：前端逐字显示 AI 回答，不得等待全部生成完成后再展示

### 3. 可选扩展功能（加分项）

* **意图识别**：在调用 LLM 前，先对用户问题做分类（如：产品咨询 / 售后问题 / 闲聊 / 投诉），并在会话记录中标注
* **追问引导**：AI 回答结束后，自动生成 2-3 个相关的追问建议供用户点击
* **管理后台**：查看全量会话记录、用户反馈统计、日均问答量折线图
* **多知识库路由**：支持创建多个知识库，提问时自动判断路由到最相关的知识库

## 技术栈要求

你可以自由选择技术栈，但我们建议使用主流方案：

**前端（二选一）**

* React + TypeScript
* Vue 3 + TypeScript

**后端**

* Python（FastAPI 或 Flask）
* 或 Java（Spring Boot），需自行集成 LLM SDK

**AI / 向量能力**

* LLM 接入：OpenAI API / 通义千问 API / 本地 Ollama（任选一种，需在文档中说明）
* 向量数据库：Chroma / Qdrant / Faiss（本地）/ Milvus（任选一种）
* Embedding 模型：可使用 OpenAI text-embedding / 本地 bge-m3 等（任选）

**数据库**

* MySQL（存储用户、会话、消息、知识库元数据）

**技术要求**

1. 前后端分离，RESTful API（或 WebSocket 用于流式输出）
2. RAG 核心链路必须自行实现（可借助 LangChain / LlamaIndex 框架，但需理解每一步逻辑）
3. 向量检索与 LLM 调用需在后端完成，前端不得直接调用 LLM API
4. 代码结构清晰，AI 相关模块需有基本的异常处理（如 LLM 超时、API 限流）
5. 流式输出：使用 SSE（Server-Sent Events）或 WebSocket 实现逐字输出

**加分项**

1. 流式输出体验流畅（逐字显示，无明显卡顿）
2. 实现了意图识别，并在会话记录中标注分类
3. Prompt 经过明显优化，能有效减少幻觉（需在文档中说明优化思路）
4. 知识库支持增量更新（新增文档不影响已有向量数据）
5. **大规模知识检索下的 LLM 执行保障**：在企业级场景中，当知识库文档非常多时，检索返回的相关片段和业务规则可能有数十条甚至更多。请设计并实现一种机制，确保 LLM 在处理大量上下文时：（1）不会因"注意力稀释"而遗漏某条关键规则；（2）不会在信息过载时产生幻觉（编造不存在的规则或混淆不同规则）。例如：对检索结果进行分层摘要、规则优先级排序、分步校验等策略。需在文档中说明你的设计思路和效果验证方式，项目代码里面实现更好
6. **终极挑战 - 改造系统实现AI Agent 的任务拆解能力**：

* 在真实的研发团队中，一个用户需求往往需要同时改动多个微服务（比如前端、用户服务、订单服务、通知服务等）。假设你有一个 AI Agent 负责根据用户需求自动编写代码，请思考并设计：当 Agent 收到一条需求（如"用户下单后自动发送短信通知"）和一整套系统的技术文档/接口文档后，它如何准确判断——这个需求需要改哪几个微服务？哪些改动可以同时进行（互不影响）？哪些改动必须按先后顺序来（有依赖关系）？你可以用流程图或示例说明你的拆解思路，不要求实现完整代码（但项目里面实现更好）

## 常见问题

**Q：必须使用付费的 LLM API 吗，我没有 API Key 怎么办？**

A：不强制要求付费 API。可使用免费额度（如通义千问 / 月之暗面 / Groq 均有免费 tier），也可使用本地 Ollama 运行开源模型（如 Llama 3 / Qwen2）。请在运行指南中说明你的模型选择和配置方式。

**Q：必须使用 LangChain 吗？**

A：不强制。你可以直接调用 LLM SDK 和向量库 SDK 手动实现 RAG 链路，这反而能更好地展示你对原理的理解。如果使用 LangChain / LlamaIndex，需要在文档中说明每个 Chain / Pipeline 的作用。

**Q：流式输出必须实现吗？**

A：是的，这是 AI 对话产品的基本体验要求，属于必做项。推荐使用 SSE（Server-Sent Events）实现，相对简单且兼容性好。

**Q：向量数据库必须部署独立服务吗？**

A：不强制。可以使用 Chroma 或 Faiss 的本地文件模式，无需启动额外服务，适合快速开发。

**Q：知识库文档格式支持哪些？**

A：必须支持 .txt 和 .md 格式。.pdf 格式为可选（需引入 PDF 解析库如 PyMuPDF）。
