# 运行指南

> AI 智能客服系统 — 本地运行完整说明

## 环境要求

| 依赖 | 最低版本 | 说明 |
|------|----------|------|
| Python | 3.12+ | 后端运行环境 |
| Node.js | 20+ | 前端构建工具链 |
| MySQL | 8.0+ | 结构化数据存储 |
| pip | 24+ | Python 包管理 |
| npm | 10+ | 前端包管理 |

> **注意**：BGE-M3 模型首次运行时会自动下载（约 2GB）。国内用户推荐从 ModelScope 下载，速度快且稳定。下载后将模型路径配置为 `EMBEDDING_MODEL=./models/bge-m3`（详见下方 BGE-M3 下载说明）。

## 1. 获取 DeepSeek API Key

1. 访问 [platform.deepseek.com](https://platform.deepseek.com/)
2. 注册/登录账号
3. 进入「API Keys」页面，点击「创建 API Key」
4. 复制 Key（形如 `sk-xxxxxxxxxxxxxxxx`）
5. DeepSeek 新用户通常有免费额度（¥10），足够开发测试

## 2. MySQL 数据库设置

### 方式 A：本地 MySQL（推荐）

```bash
# macOS (Homebrew)
brew install mysql
brew services start mysql

# Windows
# 下载安装 MySQL Community Server: https://dev.mysql.com/downloads/mysql/

# Linux (Ubuntu)
sudo apt install mysql-server
sudo systemctl start mysql
```

创建数据库：

```sql
mysql -u root -p
CREATE DATABASE ics_customer_service CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'ics_user'@'localhost' IDENTIFIED BY 'ics_password_2024';
GRANT ALL PRIVILEGES ON ics_customer_service.* TO 'ics_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

> 上述账号密码为开发示例，生产环境请使用强密码。

### 方式 B：Docker MySQL

```bash
docker run -d \
  --name ics-mysql \
  -e MYSQL_ROOT_PASSWORD=root123 \
  -e MYSQL_DATABASE=ics_customer_service \
  -p 3306:3306 \
  mysql:8.0
```

## 5a. 知识库管理

系统支持多知识库分类管理。`init_knowledge.py` 将所有示例文档存入默认知识库，你也可以手动创建分类知识库：

1. 打开 http://localhost:5173 → 左侧"知识库管理"
2. 点击"新建知识库"，创建以下 3 个建议知识库：
   - **产品知识库** — 描述：产品文档、FAQ、使用手册、版本日志
   - **售后与服务** — 描述：退换货政策、技术支持、投诉处理
   - **法律条款** — 描述：用户协议、隐私政策、合规文档
3. 上传文档时选择对应知识库（支持 .txt / .md / .pdf，≤10MB）
4. 文档上传后自动向量化，状态变为"ready"后可检索
5. 点击文档可「查看内容」和「查看分块」详情
6. 删除知识库需二次确认，关联文档的 kb_id 置空（文档和向量保留）

**对话时指定知识库**：在聊天输入框上方选择知识库，LLM 将仅检索该知识库中的内容回答问题。

## 8. 故障排查

### Q: 启动后端报 `ModuleNotFoundError: No module named 'xxx'`
A: 确保已激活虚拟环境且执行了 `pip install -r requirements.txt`

### Q: init_knowledge.py 报 `Connection refused` 或 `Access denied`
A: 检查 MySQL 是否启动，`.env` 中数据库连接信息是否正确

### Q: BGE-M3 下载失败或超时
A: 
1. **推荐方式（国内用户）**：从 ModelScope 下载模型：
   ```bash
   pip install modelscope
   python -c "from modelscope import snapshot_download; snapshot_download('BAAI/bge-m3', cache_dir='./models')"
   ```
   然后在 `.env` 中配置 `EMBEDDING_MODEL=./models/bge-m3`
2. **备选方式**：设置 HuggingFace 镜像：`export HF_ENDPOINT=https://hf-mirror.com`（国内用户推荐）

### Q: DeepSeek API 返回 401 或连接超时
A: 检查 `.env` 中 `DEEPSEEK_API_KEY` 是否正确；检查网络是否能访问 `api.deepseek.com`

### Q: 前端页面空白
A: 确认后端已启动在 8000 端口；检查浏览器控制台是否有 CORS 错误

### Q: 聊天返回"AI 服务响应超时"
A: 检查网络连接；DeepSeek API 可能繁忙，稍后重试

### Q: init_knowledge.py 报 "知识库已存在" 或文档重复
A: 默认知识库 'default' 只会创建一次。如需重新初始化：(1) 删除 `data/milvus/` 目录；(2) MySQL 中删除 `documents` 和 `knowledge_bases` 表数据；(3) 重新运行 `python init_knowledge.py`

### Q: 多知识库路由不生效
A: 确认 `agent/decomposer.py` 中的关键词规则覆盖了你的使用场景；检查 `.env` 中知识库相关配置；查看后端日志确认路由决策

### Q: 知识库文档上传后检索不到
A: (1) 确认文档已成功入库（检查前端知识库管理页面的文档列表）；(2) 确认检索关键词与文档内容相关；(3) 检查 `SIMILARITY_THRESHOLD` 阈值设置（默认 0.65），可适当降低

## 9. 常见问题 (FAQ)

**Q：必须使用付费的 LLM API 吗？**
A：不强制。DeepSeek 新用户有免费额度。也可修改 `app/rag/llm.py` 切换到其他兼容 OpenAI SDK 的 API（如通义千问、Groq 等免费 tier），或使用本地 Ollama。

**Q：必须使用 LangChain 吗？**
A：本项目不使用 LangChain。RAG 核心链路全部手动实现，仅 PDF 解析复用 LlamaIndex Reader。

**Q：向量数据库必须部署独立服务吗？**
A：不需要。本系统使用 Milvus Lite 嵌入式模式，向量数据存储在本地文件中，零额外部署。

**Q：没有 GPU 能跑吗？**
A：可以。BGE-M3 在 CPU 上推理完全可行。本项目默认使用 CPU 推理（`.env` 中 `EMBEDDING_DEVICE=cpu`）。

**Q：如何切换 LLM？**
A：修改 `.env` 中 `DEEPSEEK_BASE_URL`、`DEEPSEEK_MODEL`、`DEEPSEEK_API_KEY` 即可。兼容所有 OpenAI SDK 兼容的 API 服务。
